# 虚空塔 · 怪物设计案

## 设计目标

丰富战斗深度，打破现有 "打锈蚀哨兵 x 十几次" 的单调体验。核心改动：

1. **多敌人战斗** — 不再是1v1，多数战斗为 1v2 或 1v3
2. **楼层差异化** — 低层、中层、高层敌人不同，属性随楼层成长
3. **多阶段 AI** — 精英/BOSS 半血后切换模式，战局更动态
4. **不同敌人定位** — 攻击型、防守型、干扰型、反伤型、治疗型，组合出不同压力

## 四、楼层分布

| 楼层 | COMBAT 敌人池 | ELITE 敌人池 | HP 系数 |
|---|---|---|---|
| 1-3 层 | 锈蚀哨兵、暗影信徒、虚空爬行者 | 锈蚀哨兵·精英 | 1.0x |
| 4-6 层 | 暗影信徒、虚空爬行者、水晶尖刺 | 暗影祭司、虚空巨兽 | 1.3x |
| 7-8 层 | 虚空爬行者、水晶尖刺 | 虚空巨兽、水晶守卫 | 1.6x |

每场 COMBAT 战斗生成 **2 个敌人**（从池中随机选，可重复）。
ELITE 生成 **1 个精英**。
BOSS 从三个 BOSS 中 **三选一**。

## 五、意图类型总表

| 意图类型 | 效果 | 使用敌人 |
|---|---|---|
| `attack` | 造成伤害 | 全部 |
| `block` | 获得格挡 | 哨兵、尖刺、守卫、巨像、龙 |
| `debuff_rust` | 玩家下回合第一张牌红费+1 | 哨兵系列、巨像 |
| `steal_blue` | 偷取玩家蓝费 | 信徒、领主 |
| `heal_self` | 治疗自己 | 祭司 |
| `apply_vulnerable` | 给玩家叠易伤 | 祭司、领主 |
| `apply_weak` | 给玩家叠虚弱 | 龙 |
| `buff_strength` | 增加力量（伤害加成） | 巨兽 |
| `buff_retaliate` | 增加反伤 | 尖刺、守卫 |
| `summon` | 召唤一个随从 | 领主 |
| `attack_all` | AOE 伤害 | 领主、龙 |

## 六、虚空裂隙规则

| 规则 | 效果 | 难度 |
|---|---|---|
| `reverse` | 红蓝费互换 | ★★☆ |
| `silent` | 每打一张牌，弃牌库顶 1 张 | ★★☆ |
| `mirror` | 每打一张牌，受到其伤害 50% 的反馈 | ★★★ |
| `deplete` | 卡牌打出后移除（不进入弃牌堆） | ★★★ |
| `chaos` | 回合开始时随机交换手牌和牌库顶 | ★★★★ |

## 七、平衡说明

- **双阶段 AI**：精英和 BOSS 半血后切换至 `pattern_low_hp`，战斗节奏呈"先稳后急"的曲线
- **多敌人倍率**：1v2 战斗的有效敌人输出 ≈ 1v1 的 1.6 倍，设计时已通过单敌 HP (28-35) 和伤害 (6-10) 回调
- **楼层缩放**：4-6 层敌人 HP +30%，7-8 层 +60%，确保后期不会因玩家牌组成型而过于简单
- **裂隙规则**：主要作用于玩家的资源/牌库管理，不影响敌人数值
